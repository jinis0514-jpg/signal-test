# 장애 시 확인할 로그 포인트

## 1) 인증/권한

- 로그인 실패율, 토큰 갱신 실패율
- Supabase RLS 에러 코드(`42501`, permission denied)
- 프로필/구독 조회 실패율

## 2) 데이터/저장

- 전략 저장(create/update/delete) 실패 로그
- PDF 업로드 실패(용량, mime, 권한)
- DB 타임아웃/커넥션 에러

## 3) 실시간/시세

- 가격 fetch 실패율
- 폴링 간격별 응답시간 p50/p95
- fallback 데이터 전환 비율

## 4) 결제

- checkout 생성 실패율
- 웹훅 수신 성공/실패 건수
- 결제 성공 후 플랜 반영 지연 시간
- 중복 이벤트 처리 횟수

## 5) 사용자 영향 지표

- 화이트스크린(JS 치명 에러) 발생률
- 주요 페이지 로드 시간(Home/Signal/Market/Validation)
- 알림 발송 지연/누락 건수

## 즉시 점검 순서 (런북)

1. 최근 배포/환경변경 확인
2. 인증/DB 에러율 확인
3. 결제/웹훅 큐 적체 확인
4. 시세 API 상태 및 fallback 비율 확인
5. 영향 범위 공지 + 임시 완화(폴링 완화/캐시 고정)
